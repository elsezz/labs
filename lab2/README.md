## Опис проєкту

Лабораторна робота присвячена автоматизації збору даних, їх очищенню, трансформації та статистичному аналізу з використанням екосистеми Python (бібліотеки Pandas, SciPy, Scikit-learn). Проєкт складається з двох частин:

**Частина 1 (`lab2_1.ipynb`) — Аналіз індексів VHI (NOAA):**  
Програмна реалізація автоматизованого завантаження даних щодо стану рослинності для всіх адміністративних одиниць України. Включає обробку сирих файлів (видалення HTML-тегів, обробку пропущених значень), переіндексацію областей за українською абеткою, а також розрахунок екстремумів і середніх значень за визначені періоди.

**Частина 2 (`lab2_2.ipynb`) — Аналіз енергоспоживання (UCI Dataset):**  
Дослідження набору даних Individual Household Electric Power Consumption. Включає автоматизоване завантаження датасету, очищення даних, формування вибірок за специфічними критеріями (наприклад, за часом доби та рівнем споживання), нормалізацію та стандартизацію значень, розрахунок коефіцієнтів кореляції (Пірсона та Спірмена), а також застосування One-Hot Encoding для категоріальних змінних.

## Інструкція із запуску

1. Відкрийте термінал у кореневій директорії проєкту (наприклад, `lab_02`).

2. Створіть віртуальне середовище:
   ```bash
   python -m venv venv
   ```

3. Активуйте віртуальне середовище:

   - **Windows:**
     ```dos
     venv\Scripts\activate
     ```
   - **Linux / macOS:**
     ```bash
     source venv/bin/activate
     ```

4. Встановіть необхідні залежності з файлу `requirements.txt`:
   ```bash
   pip install -r requirements.txt
   ```

5. Запустіть середовище Jupyter Notebook:
   ```bash
   jupyter notebook
   ```

## Примітки щодо роботи з даними

У **Частині 2** набір даних має значний обсяг, проте його ручне завантаження не вимагається. Файл `lab2_2.ipynb` містить алгоритм (`download_with_retry`), який автоматично завантажує ZIP-архів з репозиторію UCI та розпаковує його під час першого виконання коду.

Аналогічно, у **Частині 1** директорія `vhi_data` створюється та заповнюється даними автоматично під час виконання коду.
