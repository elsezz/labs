## Опис роботи
Ця програма демонструє роботу з потоковим відео з веб-камери за допомогою бібліотеки OpenCV у середовищі Linux. Проєкт має модульну архітектуру (класи винесені в окремі файли) та підтримує інтерактивну обробку кадрів у реальному часі.

## Функціонал та керування
Під час роботи програми ви можете перемикати режими обробки та взаємодіяти з відеорядом:

* **Клавіатура (Режими обробки):**
  * `1` — Звичайний режим (Normal).
  * `2` — Інверсія кольорів (Invert).
  * `3` — Розмиття Гауса (Gaussian Blur).
  * `4` — Фільтр Canny (Виділення контурів).
  * `Q` або `ESC` — Завершення роботи програми та закриття камери.
* **Миша:**
  * **Слайдер "Brightness":** Регулювання яскравості кадру (від -50 до +50 від базового значення).

## Вимоги до операційної системи та заліза

* **Операційна система:** Linux (протестовано на Ubuntu та Kali Linux).
* **Апаратне забезпечення:**
  * Процесор: 1.5 GHz або швидший.
  * Оперативна пам'ять: від 1 GB.
  * Наявність підключеної веб-камери (пристрій `/dev/video0` або аналогічний).
* **Залежності для збірки:**
  * Компілятор з підтримкою C++17 (GCC/G++).
  * CMake (версія 3.10+).
  * Make.
  * Бібліотека OpenCV (версія 4.x).

## Інструкція для автоматизованої збірки та запуску

Проєкт містить зручні bash-скрипти для автоматизації налаштування на чистій системі.

Відкрийте термінал та клонуйте репозиторій. Після цього перейдіть в робочу директорію:
```bash
git clone https://github.com/elsezz/labs.git
cd labs/lab6/Lab6_OpenCV 
```

Надання права доступу скриптам (за потреби):
```bash
chmod +x preinstall.sh build.sh run.sh
```

Для встановлення залежностей скрипт автоматично встановить CMake, компілятори та бібліотеку OpenCV (потребує прав адміністратора):
```bash
sudo ./preinstall.sh
```
Крок 4. Збірка проєкту

Скрипт створить директорію build, запустить CMake та скомпілює вихідний код:
```bash
./build.sh
```

Далі скрипт запустить скомпільований виконуваний файл. (Може знадобитися запуск через sudo, якщо ваша система вимагає прав root для доступу до /dev/video0):
```bash
./run.sh
(або sudo ./run.sh)
```
